# ERB 文件格式

## 文件位置

ERB 文件放在游戏目录下的 `ERB` 文件夹内。

## 编码

Emuera 使用 **UTF-8** 编码（eramaker 使用 Shift_JIS）。

## 扩展名

- `.ERB`：脚本文件
- `.ERH`：头文件（先于 ERB 处理）

## 基本书写规则

### 注释

```
; 单行注释（分号开头）
;!; Emuera 专用有效行（eramaker 中为注释）
;#; 调试模式专用行
```

### 空行

空行被忽略。

### 行头空白

行头的半角空格和 Tab 被忽略。

### 一行完整

一个命令必须在一行内写完。使用 `{ }` 可跨行连接。

### 不要行尾注释符号

eramaker 不支持行尾 `;` 注释，Emuera 支持：

```
; Emuera 中有效
A = B ; 行尾注释
```

但 PRINT 等直接字符串参数中不行：

```
PRINT 你好; 注释  ; 会打印「你好; 注释」
```

## 特殊行标识

### [SKIPSTART] / [SKIPEND]

```
[SKIPSTART]
; eramaker 中跳过此区间
[SKIPEND]
```

与 `;!;` 配合实现跨引擎兼容：

```
;!;[SKIPSTART]
PRINTW Emuera专用代码
;!;[SKIPEND]

[SKIPSTART]
; eramaker专用代码
[SKIPEND]
```

## 文件处理顺序

1. CSV 文件夹所有 `.csv`
2. ERB 文件夹所有 `.ERH`
3. ERB 文件夹所有 `.ERB`

同一步骤内的处理顺序取决于文件名字顺。

## 函数命名规则

- 首字符不能是半角数字（不推荐）
- 符号只能用 `_`
- 不可含 `,`、`(`、`@` 等符号
- 全角字符可用
- 不能与已有命令同名

## 最佳实践

1. 使用 `SHOP.ERB`、`TRAIN.ERB` 等分文件管理
2. ERH 中定义宏和全局变量
3. 函数名使用全角或英文下划线命名
4. 重要变量在 ERH 中用 `#DIM SAVEDATA` 定义
5. 每行一个命令，避免过于复杂的嵌套
