# 表达式与语法

本文档说明 Emuera 的表达式与扩展语法特性。

## 注释

```
;这是一行注释
A = B ;把B赋值给A
```

以上展示了两种注释方法。
但行尾的注释里， PRINT 等命令的参数字符串中的 `;` 不会被解析为注释，例如：

```
PRINT foobar;示例文本   ; 打印 "foobar;示例文本"
```

**建议注释单起一行，不要写在行尾。**

> 存在几种特殊注释前缀，可在不同版本的解释器中表现出不同行为，详见 `references/eramaker-diff/differences.md`。但现在的游戏分发通常直接附带脚本解释器，因此**不建议使用它们**。

## 行连接

用 `{` 和 `}` 包围的多行内容被视为一行：

```
{
    #DIM CONST HOGE =
        1,2,3,4
}
; 等价于: #DIM CONST HOGE = 1,2,3,4
```

注意：
- `{` 和 `}` 所在行不能有其他字符（空白除外）
- 换行处会补入半角空格
- 行连接在注释解析之前处理

## 角色数组增强

Emuera 不限制角色数量（仅受内存限制）。

`chara*.csv` 匹配所有符合格式的文件，如 `chara101.csv`、`charaABC.csv` 等。

编号重叠时，先读取的优先。

## 整数范围

Emuera 使用 64 位整数（`-9223372036854775808 ～ 9223372036854775807`）。

### 常量表示法

```
X = 32        ; 十进制
X = 0b100000  ; 二进制
X = 0x20      ; 十六进制
X = 1p5       ; 1 × 2^5 = 32（位运算配合）
X = 13e3      ; 13 × 10^3 = 13000
```

- 不支持八进制（`012` 视为 `12`，不是 `10`）
- `p` / `e` 后只接受常量，不接受表达式
- `1p0`～`1p63` 常用于位掩码判定：`IF (A & 1p0)` 检查第 0 位

## 数组变量的批量赋值

```
; 向 A:10～A:12 赋值 1,2,3
A:10 = 1,2,3

; 向 DA:0:0～DA:0:2 赋值 1,2,3
DA:0:0 = 1,2,3
```

- 不可用于复合赋值（`A += 1,2,3` 等不可）
- 字符串数组需要使用字符串表达式：

```
; 将 "草莓,甜瓜,蓝色夏威夷" 赋值给 STR:20
STR:20 = 草莓,甜瓜,蓝色夏威夷

; 分别赋值给 STR:20～STR:22
STR:20 '= "草莓", "甜瓜", "蓝色夏威夷"
```

## FORM 语法（格式化字符串）

字符串变量赋值时可以使用与 `PRINTFORM` 相同的格式：

```
SAVESTR:0 = %RESULTS%
```

使用 `\` 转义：

```
SAVESTR:0 = \%RESULT\%       ; 赋值 "%RESULT%" 字面量
SAVESTR:0 = \\               ; 赋值 "\"
```

### `@"～"` 字符串表达式中的 FORM 语法

在字符串表达式中使用 FORM 语法（`{}`/`%%`）时，必须用 `@"～"` 包裹：

```
; 错误——字符串表达式中不能直接用 %%
RESULTS += %STR:0%

; 正确——用 @"～" 包裹
RESULTS += @"%STR:0%"

; 也可以用字符串连接
RESULTS += STR:0
```

`@"～"` 中若只有 `\@～\@` 三目运算符，可省略 `@"～"`。

### FORM 语法中的位数和对齐

数值展开 `{变量}` 和字符串展开 `%变量%` 均支持位数和对齐参数：

```
; 数值：{变量,显示位数,对齐}
PRINTFORML [{A,10}]       ; [    123456]  （默认右对齐）
PRINTFORML [{A,10,LEFT}]  ; [123456    ]  （左对齐）

; 字符串：%变量,显示位数,对齐%
PRINTFORML [%STR:0,10,LEFT%]  ; [あいう    ]
```

- 全角字符算 2 个字符宽度
- 对齐：`RIGHT`（默认）或 `LEFT`
- 位数不足时用半角空格填充

## 字符串表达式赋值

使用 `'=` 运算符和字符串表达式：

```
STR '= "字符串"                         ; STR = 字符串
STR '= TSTR:0 + "示例"               ; STR = %TSTR:0%示例
```

## 字符串形式的数组索引

以下变量支持用 CSV 定义的字符串名引用：

- `ITEM`（item.csv）
- `TALENT`（talent.csv）
- `ABL`（abl.csv）
- `BASE`（base.csv）
- `EXP`（exp.csv）
- `MARK`（mark.csv）
- `PALAM`（palam.csv）
- `EX`（ex.csv）
- `STAIN`（stain.csv）
- `SOURCE`（source.csv）
- `CFLAG`（cflag.csv）
- `FLAG`（flag.csv）
- `TFLAG`（tflag.csv）
- `STR`（strname.csv）
- `TSTR`（tstr.csv）
- `CSTR`（cstr.csv）
- `EQUIP`（equip.csv）
- `TEQUIP`（tequip.csv）

```
; 数值索引
ABL:MASTER:0 = 10

; 字符串索引
ABL:MASTER:"体力" = 10
```

### 名称冲突优先级规则

当 CSV 定义的名称与用户定义变量同名时：

1. **用户定义变量优先**：假设在 ABL.csv 中定义了 `战斗等级`，那么如果用 `#DIM 战斗等级, 0` 定义了变量 `战斗等级`，则 `ABL:战斗等级` 被解释为该变量的值（而非 CSV 中名为 "战斗等级" 的索引）
2. **数值优先**：如果 `abl.csv` 中有 `0,10` 这样的条目（名称是数字字符串），`ABL:10` 始终被解释为数值索引 `10`（而非名为 "10" 的 CSV 条目）
3. **`RELATION` 例外**：`RELATION` 的字符串索引引用 `NAME` 或 `CALLNAME`，而非专门的 CSV

## 三目运算符

### 数值型

```
X = condition ? value_if_true # value_if_false
```

等价于：

```
IF condition
    X = value_if_true
ELSE
    X = value_if_false
ENDIF
```

可嵌套在表达式中：`Y = (X > 0 ? X # -X) * 2`

### 字符串型

```
STR '= \@ condition ? "true_string" # "false_string" \@
```

字符串型三目运算中 `#` 不可省略。

## 自增 / 自减

```
A++     ; A = A + 1
A--     ; A = A - 1
```

不能与其他运算符组合使用。
