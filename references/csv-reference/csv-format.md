# CSV 格式参考

本文档说明 ERA Basic 中使用的各种 CSV 文件格式。

## CSV 基本规格

- 编码：UTF-8
- 扩展名：`.CSV`
- 位置：放在 `CSV` 文件夹内
- 分隔符：半角逗号 `,`
- 数值必须使用半角字符
- 字符串不要用 `""` 括起
- 以 `;` 开头的行为注释行
- 空行被忽略

## 主要 CSV 文件一览

### 角色定义（chara*.csv）

角色 CSV 文件名格式为 `chara*.csv`（如 `chara0.csv`、`chara01.csv`、`charaABC.csv`），放在 `CSV` 文件夹中。所有匹配的文件都会被加载，编号重叠时先读取的优先。

#### 逐字段格式

每个字段占一行或若干行，字段之间按出现顺序排列：

| 字段关键字 | 格式 | 对应变量 | 说明 |
|-----------|------|---------|------|
| `番号` | `番号,数值` | `NO` | 角色定义编号（必填） |
| `名前` | `名前,名称` | `NAME` | 角色名称 |
| `呼び名` | `呼び名,称呼` | `CALLNAME` | 角色称呼 |
| `基礎` | `基礎,编号,数值` | `BASE` | 基础值（可多行） |
| `能力` | `能力,编号,数值` | `ABL` | 能力值（可多行） |
| `素質` | `素質,编号` | `TALENT` | 天赋（仅编号，值为 1，可多行） |
| `経験` | `経験,编号,数值` | `EXP` | 经验值（可多行） |
| `相性` | `相性,编号,数值` | `RELATION` | 相性值（可多行，编号为对方角色定义编号） |
| `助手` | `助手,0或1` | `ISASSI` | 是否为助手 |
| `フラグ` | `フラグ,编号,数值` | `CFLAG` | 角色标志（可多行） |

字段可以填日语，也可以填变量名。
编号可以填数字，也可以填定义的字段名。

#### 示例

```
番号,0
名前,铃仙
呼び名,铃仙
基礎,0,1000
基礎,1,800
能力,0,5
能力,1,3
素質,0
素質,5
経験,0,10
相性,1,120
フラグ,0,50
```

假设有 `CFLAG.csv` 文件，内容如下：

```
0,被告白
```

那么角色文件中也可以直接写：

```
FLAG,被告白,1
```

这样可以增加可读性，推荐这样写。

### 能力名称（abl.csv）

```
0,体力
1,气力
2,腕力
...
```

用于定义 `ABL` 的元素名。`ABLNAME` 可引用。

> **空号规则**：`abl.csv`、`palam.csv`、`mark.csv` 推荐不留空号（连续编号），`talent.csv`、`exp.csv`、`train.csv`、`item.csv`、`str.csv` 允许留空号。

### 素质名称（talent.csv）

```
0,处女
1,敏感
2,痛觉
...
```

用于定义 `TALENT` 的元素名。`TALENTNAME` 可引用。

### 经验名称（exp.csv）

```
0,接吻
1,口交
...
```

用于定义 `EXP` 的元素名。`EXPNAME` 可引用。

### 参数名称（palam.csv）

```
0,快感
1,疼痛
...
```

用于定义 `PALAM`、`JUEL`、`GOTJUEL`、`UP`、`DOWN`、`CUP`、`CDOWN` 的元素名。

### 刻印名称（mark.csv）

```
0,阴蒂责罚
1,震动棒
...
```

用于定义 `MARK` 的元素名。`MARKNAME` 可引用。

### 基础名称（base.csv）

```
0,体力
1,魅力
2,教养
...
```

用于定义 `BASE`、`MAXBASE`、`LOSEBASE`、`DOWNBASE` 的元素名。

### 调教名称（train.csv）

```
0,爱抚
1,接吻
...
```

用于定义调教命令的名称。`TRAINNAME` 可引用。

### 物品（item.csv）

```
0,药草,200
1,解毒药,500
...
```

格式：`编号,名称,价格`

`ITEMNAME` 和 `ITEMPRICE` 可引用。

### 标志名称（flag.csv）

```
0,事件A已完成
1,事件B已完成
...
```

用于定义 `FLAG` 的元素名。最大 10000 元素。

### 字符串（str.csv）

```
0,你好
1,再见
...
```

定义 `STR` 的**值**（不是名称）。
`STRNAME` 由 `strname.csv` 定义。

#### Str.csv 特殊替换符号

`str.csv` 中可以使用以下特殊符号，运行时自动替换：

| 符号 | 替换为 |
|------|--------|
| `***` | 调教中角色名（`NAME:TARGET`） |
| `+++` | 主角呼名（`CALLNAME:MASTER`） |
| `$$$` | 调教中角色呼名（`CALLNAME:TARGET`） |
| `///` | 助手名（`NAME:ASSI`） |
| `===` | 当前调教者呼名（`CALLNAME:PLAYER`） |

> 这些符号在旧项目代码中可能会遇到，但由于可读性很差，写新的代码的时候不要这样做，建议直接在 ERB 中用变量拼接。

### 字符串名称（strname.csv）

```
0,问候
1,告别
...
```

定义 `STR` 的**元素名**。最大 20000 元素。

**注意**：`str.csv` 和 `strname.csv` 是不同的文件！

### 角色字符串名称（cstr.csv）

定义 `CSTR` 的元素名。`CSTRNAME` 可引用。

### 回合字符串名称（tstr.csv）

定义 `TSTR` 的元素名。`TSTRNAME` 可引用。

### 存档字符串名称（savestr.csv）

定义 `SAVESTR` 的元素名。`SAVESTRNAME` 可引用。

### 角色 CSV 数据名称（cflag.csv）

```
0,恋爱度
1,信頼度
...
```

定义 `CFLAG` 的元素名。`CFLAGNAME` 可引用。

## gamebase.csv

游戏基础信息文件：

```
作者,サークル名
追加情報,追加テキスト
製作年,2024
タイトル,ゲームタイトル
コード,12345
バージョン,100
バージョン違い認める,1
最初からいるキャラ,1
アイテムなし,0
ウィンドウタイトル,カスタムタイトル
```

对应 `GAMEBASE_*` 系列变量。

#### 字段语义详解

| CSV 列 | 变量 | 语义说明 |
|---------|------|---------|
| `コード` | `GAMEBASE_GAMECODE` | 游戏识别码，用于存档识别。不同游戏应使用不同值 |
| `バージョン` | `GAMEBASE_VERSION` | 版本号。显示时除以 1000（如 `1824` 显示为 "1.824"） |
| `バージョン違い認める` | `GAMEBASE_ALLOWVERSION` | 设为 `1` 时允许读取不同版本的存档（跨版本兼容） |
| `最初からいるキャラ` | `GAMEBASE_DEFAULTCHARA` | 初始角色编号，对应 `ADDDEFCHARA` 添加的角色 |
| `アイテムなし` | `GAMEBASE_NOITEM` | "无道具"模式，0 和 1 之外也可以指定 |
| `ウィンドウタイトル` | `WINDOW_TITLE` | 自定义窗口标题，未设置时从 `タイトル` 和 `バージョン` 自动生成 |

## VariableSize.csv

改变内建变量的元素个数：

```
变量名,元素数
FLAG,5000
TFLAG,2000
CFLAG,2000
```

- 设定 `-1` 可禁止该变量使用
- 禁止后，赋值和引用会报错
- 系统需要时返回 `-1`
- `COUNT` 禁止后 `REPEAT` 也不可使用

## ERD 文件（变量数组元素命名）

EE+EM 扩展功能。`.ERD` 文件为 ERH 中用 `#DIM`/`#DIMS` 定义的多维变量数组提供字符串名称映射。

- 文件位置：`CSV` 文件夹，扩展名 `.ERD`
- 文件名需与变量名对应
- 多维变量用 `@1`、`@2` 后缀区分维度

### 示例

假设 ERH 中定义了：

```
#DIM CHARADATA SAVEDATA CDFLAG, 10, 10
```

则创建两个 ERD 文件：

**`CDFLAG@1.ERD`**（第一维名称）：
```
0,体力
1,气力
2,知力
```

**`CDFLAG@2.ERD`**（第二维名称）：
```
0,好感度
1,依赖度
2,服从度
```

使用时：

```
; 用字符串索引访问
CDFLAG:MASTER:体力:好感度 = 100
; 等价于 CDFLAG:MASTER:0:0 = 100

; GETNUM 查找索引（第3参数指定维度）
X = GETNUM(CDFLAG, 体力, 1)   ; X = 0
Y = GETNUM(CDFLAG, 好感度, 2)  ; Y = 0
```

### 注意事项

- ERD 功能需在配置中启用（`ERD機能を利用する`）
- 一维变量不需要 `@1` 后缀，直接用 `变量名.ERD`
- `ERDNAME` 式中函数可获取 ERD 定义的名称

## Alias 别名文件（.als）

EE+EM 扩展功能。`.als` 文件允许 CSV 变量拥有多个识别子（别名）。

- 文件位置：`CSV` 文件夹，扩展名 `.ALS`
- 文件名需与对应变量名相同

### 示例

**`ABL.ALS`**：
```
0,体力,スタミナ
1,気力,メンタル
```

这允许 `ABL:MASTER:"体力"` 和 `ABL:MASTER:"スタミナ"` 都指向 `ABL:MASTER:0`。

### 用途

- 同一概念有多种名称时（如日语/英语双名、简称/全称）
- 旧版本代码兼容：别名映射新旧名称
