# 游戏配置

## emuera.config

`emuera.config` 是 Emuera 的配置文件，位于游戏目录下（与 Emuera.exe 同级）。

首次启动时自动生成默认配置，可通过 Emuera 菜单「帮助 → 设置」修改，也可以直接用文本编辑器编辑（下次启动生效）。

### 配置文件优先级（三级覆盖系统）

Emuera 按以下顺序读取配置文件，**后读取的覆盖先读取的**：

```
csv/_default.config   →   emuera.config   →   csv/_fixed.config
```

| 文件 | 位置 | 作用 |
|------|------|------|
| `_default.config` | `csv/` 文件夹 | 仅当 `emuera.config` 不存在时生效，用于提供推荐的初始配置 |
| `emuera.config` | exe 同级目录 | 正常配置文件，用户可通过菜单修改 |
| `_fixed.config` | `csv/` 文件夹 | 强制配置，覆盖 `emuera.config`，**且对应项无法通过菜单修改** |

- **`_fixed.config`**：用于锁定对游戏运行至关重要的配置项（如行折返、背景色等）。只应设置必要的最小项，过度使用会剥夺用户的自定义权利。
- **`_default.config`**：用于推荐但非强制的初始值。用户可通过菜单覆盖。

> ⚠️ 文件名必须完全匹配（含下划线），放在子文件夹中或改名均无效。

## _replace.csv

`_replace.csv` 放在 `csv/` 文件夹内，用于自定义游戏中的显示文字和默认数值。

文件格式：每行一个项目，格式为 `项目名,值`。

### 各配置项详细说明

#### お金の単位（货币单位）

`PRINT_SHOPITEM` 显示商品价格时附加在数字上的单位符号。标准为 `$`。

- 支持全角字符（如 `円`、`元`），也支持多个字符组成的字符串（如 `金币`、`GOLD`）。
- 显示位置由下方的「単位の位置」决定。

**示例**：设为 `円` 且单位位置为 `後` → `500円`；设为 `€` 且单位位置为 `前` → `€500`。

---

#### 単位の位置（单位位置）

控制货币单位是在价格数字之前还是之后显示。取值为 `前` 或 `後`，标准为 `後`。

- `前`：单位在数字前方（如 `$500`）。
- `後`：单位在数字后方（如 `500円`）。

---

#### 起動時簡略表示（启动时简化显示）

当 Emuera 配置中的「ロード時にレポートを表示する」（加载时显示报告）功能设为 **关闭** 时，在启动阶段替代报告显示的字符串。标准为 `Now Loading...`。

- 关闭该功能后，游戏启动时不会展示加载报告，而是直接显示这里指定的文字。
- 可用于显示自定义的加载提示，如 `读取中…` 等。

---

#### 販売アイテム数（出售商品数量）

控制 `SHOP` 指令中，输入什么范围的编号会进入商品购买/`@EVENTBUY` 处理流程，超出范围则调用 `@USERSHOP`。

- 标准行为（era 原版）：`PRINT_SHOPITEM` 会显示所有 `ITEMSALES` 不为 0 的 `ITEMNAME`（0～999），但只有输入 `0～99` 时才进入购买处理与 `@EVENTBUY`，输入其他值则调用 `@USERSHOP`。
- Emuera 基本重现了这一逻辑。修改「販売アイテム数」后，购买处理的号码范围随之改变：
  - 输入 **0 ～（販売アイテム数 − 1）**：进入商品购买流程（无论是否购买成功，**都不会调用 `@USERSHOP`**）。
  - 输入 **負值 或 ≥ 販売アイテム数**：调用 `@USERSHOP`。
- 例如，设为 `1000`：输入 0～999 进入购买，1000 以上或负数触发 `@USERSHOP`。
- ⚠️ **注意**：若「販売アイテム数」设为 1000，输入 `[200] - 保存` 或 `[300] - 读取` 等自定义菜单项时，会被当成「购买 200 号 / 300 号商品」来处理，**不会**触发 `@USERSHOP`。也就是说，自定义菜单数字必须避开 0～(N−1) 范围。
- **与 VariableSize.csv 的关系**：`PRINT_SHOPITEM` 实际显示的物品数量取决于 `VariableSize.csv` 中 `ITEMNAME` 和 `ITEMSALES` 数组的元素数——取两者中 **较小者** 作为显示范围。如需让商店展示更多/更少物品，请修改 `VariableSize.csv` 中对应数组的尺寸。

---

#### DRAWLINE文字（分隔线文字）

`DRAWLINE` 命令绘制分隔线时使用的字符。标准为 `-`。

- `DRAWLINE` 输出的线条由该字符重复构成。
- 可替换为其他字符（如 `=`、`─` 等）来改变视觉效果。

---

#### BAR文字1 / BAR文字2（进度条字符）

`BAR` 或 `BARL` 命令绘制进度条时使用的两种字符：

- **BAR文字1**（标准 `*`）：表示已填充部分。
- **BAR文字2**（标准 `.`）：表示未填充部分。

效果类似 `[****....]` 的可视化条状图。

- 支持全角字符或多字符字符串（如 `█` / `░`），但需注意显示可能偏移对齐。
- ⚠️ **重要**：如果指定为**数字**，Emuera 会将诸如 `[88881111]` 等输出误解为可点击按钮，引发异常行为。请务必使用非数字字符。

---

#### システムメニュー0 / システムメニュー1（系统菜单选项）

标题画面初始显示的两个选项文字：

- **システムメニュー0**（标准 `最初からはじめる`）：选项 0 → 「从头开始」
- **システムメニュー1**（标准 `ロードしてはじめる`）：选项 1 → 「读取存档」

显示效果如下：
```
[0] 最初からはじめる
[1] ロードしてはじめる
```

- ⚠️ **注意**：这两个设置**仅在未自定义 `@SYSTEM_TITLE` 时生效**。如果在代码中定义了 `@SYSTEM_TITLE` 来实现自定义标题画面，则系统菜单选项文字完全由 `@SYSTEM_TITLE` 内部的显示逻辑负责，`_replace.csv` 中的设定不再自动应用。

---

#### COM_ABLE初期値（COM 可执行初始值）

在使用 `TRAIN` 命令时，若未找到对应编号的 `@COM_ABLE{X}` 事件，则使用此值来决定该 COM 是否可执行。

- `1`（默认）：未定义 `@COM_ABLE` 时，视为**可执行**。
- `0`：未定义 `@COM_ABLE` 时，视为**不可执行**。

例如，假设没有定义 `@COM_ABLE3`：设为 1 时 COM3 可执行；设为 0 时 COM3 不可执行。

---

#### 汚れの初期値（污秽初始值）

`STAIN` 数据初始化时各元素的默认值。用 `/`（半角斜线）分隔每个元素。

标准值为 `0, 0, 2, 1, 8`，含义：

- `STAIN:0` = 0
- `STAIN:1` = 0
- `STAIN:2` = 2
- `STAIN:3` = 1
- `STAIN:4` = 8

可继续追加 `/` 后的值来覆盖更多元素。

---

#### 時間切れ表示（超时显示文字）

当 `TINPUT` 等有时间限制的输入命令超时时，显示给玩家的提示文字。标准为 `時間切れ`。

- 适用于 `TINPUT`、`[WAIT]` 带时间限制等所有限时输入场景。
- 可自定义为其他文字，如 `时间到`、`Time's up!` 等。

---

#### EXPLVの初期値（经验等级初始值）

`EXPLV`（经验等级阈值）的初始化值。用 `/` 分隔每个元素。

标准值为 `0, 1, 4, 20, 50, 200`：

- `EXPLV:0` = 0
- `EXPLV:1` = 1
- `EXPLV:2` = 4
- `EXPLV:3` = 20
- `EXPLV:4` = 50
- `EXPLV:5` = 200

可继续追加 `/` 后的值来定义更多等级。

---

#### PALAMLVの初期値（能力等级初始值）

`PALAMLV`（能力参数等级阈值）的初始化值。用 `/` 分隔每个元素。

标准值为 `0, 100, 500, 3000, 10000, 30000, 60000, 100000, 150000, 250000`：

- `PALAMLV:0` = 0
- `PALAMLV:1` = 100
- `PALAMLV:2` = 500
- …以此类推。

可继续追加 `/` 后的值来定义更多等级。

---

#### PBANDの初期値（玩家等级带初始值）

`PBAND:0` 的初始化值。标准为 `4`。

- `PBAND` 是一个容量限制型数组（size 由游戏定义），这里只设定 `PBAND:0` 这一项的初始值。
- 其他元素（`:1`、`:2` …）可通过代码初始化。

---

#### RELATIONの初期値（相性默认值）

当 `Chara**.csv`（角色数据文件）中未明确指定某两个角色之间的 `RELATION` 值时，使用此默认值作为相性值。标准为 `0`。

- 例如，`Chara01.csv` 中如果没有定义与角色 03 的 `RELATION`，则该关系值将被初始化为这里设定的数值。
- 设为 0 表示默认无特殊相性（中立关系）。

## _Rename.csv

文件名重定向文件，放在 `csv/` 文件夹内。

```
原名,1,新名,1
```

可将 CSV 和 ERB 中的关键字替换为其他文字（用于跨语言等场景）。

Emuera 默认不启用此功能，需在设置中勾选「使用_Rename.csv」。
