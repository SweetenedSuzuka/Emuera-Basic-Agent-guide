# Emuera 使用方法

## 运行环境

需要 .NET 7.0 以上版本的运行时环境。

> 旧版 Emuera 依赖 .NET Framework 4.5，该版本已停止支持。现已统一到 .NET 7.0。

Windows 10 / Windows 11 均确认可用。Windows XP、Vista 已不再支持。

如果启动时出现「无法正确初始化应用程序」(0xc0000135) 错误，说明未安装 .NET 7.0 运行时。
请前往 [.NET 官方下载页面](https://dotnet.microsoft.com/zh-cn/download/dotnet/7.0) 下载并安装 **.NET Desktop Runtime 7.0**。

---

## 基本使用方法

将 `EmueraXXX.exe` 放入游戏目录中，然后运行即可。

> `XXX` 部分为版本号数字。另外，根据系统设置，`.exe` 扩展名可能不显示。

启动后会自动生成 `emuera.config`（配置文件）；如果异常退出，还会生成 `emuera.log`（日志文件），可用来排查错误原因。

如果无法启动，请参见上方「运行环境」一节确认环境是否正确。

---

## 基本操作

Emuera 支持以下操作方式：

- **选择选项**：选项中会用方括号 `[0]` ～ `[99]` 标记数字，用鼠标点击对应数字即可选择。
- **跳过**：按 `Esc` 键或鼠标右键可以跳过当前输出，直接跳到下一个选项。
  - 按住鼠标右键不放再点击选项，则会在选择的同时跳到下一个选项（一步完成选择+跳过）。
- **粘贴**：按 `Ctrl + V` 可将剪贴板内容粘贴到输入框。
- **历史输入**：按上下方向键可以调出之前输入过的内容。

> Emuera 也可以直接运行通过 `EraMakerEx` 做过字符串转换后的游戏文件。
> 但该功能默认关闭；如需启用，请在菜单 **帮助 → 设置** 中勾选「**_Rename.csv を利用する**」（使用 _Rename.csv）。

## 键盘宏

Emuera 提供了简易的键盘宏功能，可以通过预定义的输入序列来自动化重复操作。

### 基本语法

| 记号 | 含义 |
|------|------|
| `\e` | 相当于按 `Esc` 键或鼠标右键 → **跳过当前输出，进入下一个选项** |
| `\n` | **输入分隔符**：将一次输入拆分为多次（模拟逐次按回车） |
| `( ... )*n` | **重复**：将括号内的输入序列重复执行 `n` 次 |

### 示例

```
0\e\n1\e\n
```
→ 输入 `0` → 跳过当前输出 → 输入 `1` → 跳过当前输出。

```
(0\e\n)*3
```
→ 等价于 `0\e\n0\e\n0\e\n`，即重复三次「输入 0 然后跳过」。

### 调用与保存

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 调用宏 | `F1` ～ `F12` | 按下后自动输入已保存的宏内容 |
| 保存宏 | `Shift + F1` ～ `Shift + F12` | 将**当前输入框中的文字**保存到对应 F 键 |

> 此功能需要 Emuera 配置中启用「**キーボードマクロを使用する**」（使用键盘宏）才可生效。

### 宏分组

按 `Ctrl + 0` ～ `Ctrl + 9` 可以切换宏分组（共 10 组，编号 0 ～ 9）。

- Emuera 启动时默认选中分组 0。
- 不同分组下的 `F1` ～ `F12` 可以分别保存**不同的宏**，因此总共可以存储 **120 个宏**（10 组 × 12 键）。
- 也可以通过输入框右键菜单切换分组。

### 中断宏执行

宏执行过程中按 `Esc` 键可以**立即中止**宏的继续执行。

### macro.txt 文件格式

保存的宏会存储在游戏目录下的 `macro.txt` 文件中。你也可以直接用文本编辑器编辑此文件来定义宏。

**文件格式**：

```plaintext
マクロキーF1:这里是按下 F1 时要输入的宏内容
マクロキーF2:这里是按下 F2 时要输入的宏内容
...
マクロキーF12:这里是按下 F12 时要输入的宏内容
```

**带分组的格式**：

```plaintext
G0:マクロキーF1:分组 0 的 F1 宏
G0:マクロキーF2:分组 0 的 F2 宏
G1:マクロキーF1:分组 1 的 F1 宏
G1:マクロキーF2:分组 1 的 F2 宏
...
```

其中：
- `G0:` ～ `G9:` 表示宏分组编号。
- `マクロキーF○:` 冒号右侧的内容即是按下 `F○` 键时要输入的宏文本。
- 修改 `グループ×:マクログループ×に設定` 冒号右边的文字可以自定义切换分组时的**提示文字**。

## 菜单功能

> 此功能需要在配置中启用「**メニューを使用する**」（使用菜单）。默认已启用。

### 文件

#### 重启（再起動）
重新启动 Emuera。

> ⚠️ 如果尚未存档，当前游戏进度会丢失，请务必注意。

#### 保存日志（ログを保存する）
将历史日志保存为指定名称的文本文件。

> 保存的最大行数与历史日志的行数一致。

#### 返回标题画面（タイトル画面へもどる）
返回游戏标题画面。

> ⚠️ 如果尚未存档，当前游戏进度会丢失，请务必注意。

#### 重新读取代码（コードを読み直す）
保留当前游戏状态不变，重新加载所有 `.erb` 文件。

- 已经调用过的函数仍会执行重新加载前的旧代码。
- 只有当旧函数不再被任何地方调用时，旧代码才会从内存中释放。
- **注意**：连续使用此功能可能导致内存不足，因为旧代码在函数释放前会一直驻留在内存中。

#### 退出（終了）
退出 Emuera。

### 帮助

#### 设置（設定）
打开配置设置面板。各配置项的详细说明请参见 [config.md](../game-config/config.md)。

## _Rename.csv 功能

> 该功能默认关闭。如需使用，请在菜单 **帮助 → 设置** 中勾选「**_Rename.csv を利用する**」。

`_Rename.csv` 位于游戏目录下，用于在 Emuera 启动时对游戏文件进行**路径重映射**。

典型用途：
- 将 era 游戏原本读取的指定路径文件替换为另一个自定义文件。
- 在不修改原 `.erb` 代码的情况下实现资源替换。

格式为 CSV，每行一条重命名规则：

```csv
原路径,新路径
```

例如：

```csv
resources\image\face1.png,resources\custom\my_face1.png
resources\bgm\bgm04.wav,resources\bgm\bgm04_remix.wav
```

Emuera 在读取文件时会先查 `_Rename.csv`，如果命中了重命名规则，就用新路径替代原路径。

> 此功能并非 Emuera 独有，`EraMakerEx` 也支持类似机制。启用后，Emuera 可以直接运行经过 `EraMakerEx` 字符串转换的游戏文件，无需额外处理。
