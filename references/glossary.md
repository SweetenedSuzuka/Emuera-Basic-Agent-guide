# 术语表

本文汇总 Emuera 相关术语。随着 Emuera 的扩展，自定义术语逐渐增多，故整理于此作为参考。

## 启动模式

### 普通模式（通常モード）

正常启动 Emuera 时的模式。双击 `EmueraXXXX.exe` 即进入此模式。

### 解析模式（解析モード）

通过命令行参数传入要解析的文件名来启动时的模式。将文件拖放到 `EmueraXXXX.exe` 上即进入此模式。解析模式下会对拖放的文件进行语法检查，因此可以用于确认开发中文件的语法错误。不过由于检查的是元素字符串形式的指定，需要准备所用变体（variant）的 CSV 文件夹。建议将文件拖放到所用变体对应的 `EmueraXXXX.exe` 上。

另外，从 `1738g` 版本起，解析模式下若启用了"加载时显示报告"（ロード時にレポートを表示する），会在每个被加载的 ERB 文件中显示其中使用的函数列表。若函数过多导致日志被刷掉，请在配置中增加日志行数。

### 调试模式（デバッグモード）

通过命令行参数传入 `-Debug` 来启动时的模式。详见[调试模式](debug.md)。

## 窗口/对话框

### 主窗口（メインウインドウ）

正常启动时最先打开的窗口。

### 主控制台（メインコンソール）

主窗口中负责输入输出的部分。

### 调试窗口（デバッグウインドウ）

在调试模式下启动时可以打开的窗口。

### 调试控制台（デバッグコンソール）

调试窗口中选择"控制台"（コンソール）标签时显示的、负责输入输出的部分。

### 设置对话框（設定ダイアログ）

从主窗口菜单的"帮助 → 设置"（ヘルプ → 設定）打开的对话框。

### 剪贴板对话框（クリップボードダイアログ）

在主控制台中按 Ctrl+C 打开的对话框。

## 函数相关

### 命令（命令）

`PRINT`、`WAIT` 等即命令。

### 函数（関数）

在 ERB 脚本中通过 `@～～` 语句定义名称，并由 `CALL` 命令等调用的东西。也包括可在表达式中使用、且通过 `@～～` 语句定义名称的函数。

### 事件函数（イベント関数）

函数名以 `EVENT` 开头、由系统调用的函数。多重定义时会全部被调用。

### 预处理器（プリプロセッサ）

ERB 文件夹内的文件中，先于其他命令处理的行。即以 `#` 开头的行，或由 `[]` 括起来的词组构成的行。以 `#` 开头的行分为"属性"和"定义"两类，详见下文。由 `[]` 括起来的词组构成的行表示特殊的代码块。

### 属性（属性，预处理器）

对函数指定的、以 `#` 开头的行，是决定函数规格和行为的预处理器。包括用于控制事件函数执行时行为的 `#PRI`、`#LATER`、`#SINGLE`、`#ONLY`，以及用于指定表达式中函数类型的 `#FUNCTION`、`#FUNCTIONS`。

### 定义（定義，预处理器）

以 `#` 开头的行，用于定义变量名称及其规格的预处理器。对函数指定的有：指定 `LOCAL`、`LOCALS` 变量元素数的 `#LOCALSIZE`、`#LOCALSSIZE`；定义变量的 `#DIM`、`#DIMS`；在 ERH 中定义 DEFINE 宏的 `#DEFINE`。

### 表达式中可用的函数（式中で使える関数）

可以从表达式中调用的"函数"。这里的"函数"不仅包括上述函数，还包括后述的内置函数（組み込み関数）。在很多编程语言中这就是通常意义上的"函数"。

### 表达式中函数（式中関数）

"表达式中可用的函数"的简称。与编程语言中的表达式中函数（匿名函数）或内联函数（inline function）无关。

### 内置函数（組み込み関数）

表达式中函数中，由 Emuera 原生内置、无需 `@～～` 定义即可使用的函数。如 `ABS(X)`、`GETTIME()` 等。容易混淆的是，它并不属于上述"函数"的定义范畴。可以将其理解为"可在表达式中使用的命令"。

### 用户定义函数（ユーザー定義関数）

用户定义的函数。即在 ERB 脚本中通过 `@～～` 语句定义名称，并由 `CALL` 命令等调用的函数。与上述"函数"是同一概念。

### `#FUNCTION(S)` 函数（`#FUNCTION(S)`関数）

通过 `@～～` 语句定义名称、且带有 `#FUNCTION` 或 `#FUNCTIONS` 属性的函数。是同时作为"函数"和"表达式中函数"的函数。

## 行/语句/表达式

### 行（行）

从换行符到下一个换行符之间的内容。编程相关语境中也称为物理行（物理行）。容易混淆的是，在编辑器相关语境中常将此称为逻辑行（論理行）。

### 语句（文）

语句（或称逻辑行）是 Emuera 中的一个处理单元。大多数语句由一个命令及其参数，或者由变量、赋值运算符和表达式组成。ERB 中遵循"一行一语句"的铁则，因此行和语句几乎同义。本文档也不特别区分二者。

### 表达式（式）

变量、常量、表达式中函数、非赋值运算符、括号及其组合。赋值运算符只能作为赋值语句的第一个运算符使用，不能在表达式中使用。

### 数值表达式（数式）

求值结果（运算结果）为数值的表达式。例如 `A+B`、`STR == "あいう"` 等。

### 字符串表达式（文字列式）

求值结果（运算结果）为字符串的表达式。例如 `STR + STR:1`、`"あ" * 10` 等。

## 变量相关

### 伪变量（擬似変数）

像 `RAND`、`CHARANUM` 这样，写法上像变量但实质上不是变量的东西。内部行为上接近表达式中函数。

### 数组变量（配列変数）

拥有多个元素的变量。数组变量的元素数可以通过 `VariableSize.csv` 更改，通常在脚本中不会增减，但部分局部变量（ローカル変数）可以在脚本中指定数组元素数。

### 角色变量（キャラクタ変数）

记录角色状态的变量。与 C 语言的 `char` 型变量无关。由于其性质，会随着 `ADDCHARA` 或 `DELCHARA` 导致的角色增减而改变元素数。因其元素指定方式（如 `NO:TARGET`）与数组变量形式相同，本文档将角色变量与数组变量加以区分。

### 二维数组变量（二重配列変数）

既是角色变量又是数组变量的变量。如 `CFLAG:TARGET:2`，接受两个参数（可省略）。由于是角色变量，第一个参数表示角色编号。同时，随着角色的增减，第一维的元素数也会增减。第二维的元素数只能通过 `VariableSize.csv` 更改，脚本中不会增减。

本文档中不使用"二重配列"这一术语，而改用"既是角色变量又是数组变量"的说法。

### 多维数组变量（多次元配列変数）

如 `DITEMTYPE` 等二维数组变量以及 `TA` 等三维数组变量。如 `DA:0:1` 或 `TA:1:2:3`，接受两个或三个参数。多维数组变量的元素数只能通过 `VariableSize.csv` 更改，脚本中不会增减。此外，多维数组变量的参数不可省略。

### 角色多维数组变量（キャラクタ多次元配列変数）

既是角色变量又是多维数组变量的变量。如 `CFLAG:TARGET:0:2`，接受三个参数（不可省略）。由于是角色变量，第一个参数表示角色编号。同时，随着角色的增减，第一维的元素数也会增减。第二维、第三维的元素数只能通过 `VariableSize.csv` 更改，脚本中不会增减。

截至 ver1807，符合此定义的变量仅有 `CDFLAG`。详见 `CDFLAG` 的相关文档。

### 局部变量（ローカル変数）

`LOCAL`、`LOCALS` 以及私有变量（プライベート変数）等，按函数（函数名）分别准备的变量。关于私有变量请参见另项。对于私有变量以外的 `LOCAL` 和 `LOCALS`，实际上并非通常意义上的局部变量，而是名为 `LOCAL@函数名` 或 `LOCALS@函数名` 的 `public static` 变量。即使退出函数，值仍会保留，且可以从函数外部进行赋值和引用。此外，递归调用等多次调用的情况下，值也会被共享。

### 全局变量（广义）（広域変数）

除 `LOCAL`、`LOCALS` 和私有变量等之外的大多数变量，所有函数共享其值。相当于一般编程语言中全局变量（global variable）的概念。此外，在 ERH 中使用 `#DIM` 或 `#DIMS` 可以定义全局变量（广义）。详见头文件（ERH）的相关文档。

### 全局变量（グローバル変数）

可以在不同存档之间共享值的变量。全局变量也属于全局变量（广义）的范畴。其保存/加载和初始化的时机与普通变量不同。Emuera 的全局变量与一般编程语言中全局变量（global variable）的概念无关。

### 私有变量（プライベート変数）

按函数（函数名）通过 `#DIM` 或 `#DIMS` 定义的变量。这些也属于局部变量，因此每个函数各自独立。但与同为局部变量的 `LOCAL` 等不同，无法通过 `@函数名` 等方式从函数外部进行赋值或引用。详见用户定义变量的相关文档。
