# 调试功能

> ⚠️ **不推荐在开发中使用**。本文档仅为 Emuera 完整参考而列出调试功能，**不应作为日常开发的推荐工具**。
>
> 原因：
> 1. `;#;` 前缀和 `[IF_DEBUG]` 块在非调试模式下完全不可见——AI 若用这些语法编写新模块，正常游戏中模块不会执行。
> 2. 将调试代码转正需逐行移除 `;#;` 前缀，在 AI 生成代码时，这一步会造成极大的时间和词元浪费。
> 3. ERB 没有高级语言那样的断点/单步/条件断点等调试方式，不如和命令行软件一样直接在关键处打印所需信息（就像大学新生写的那些一样）。
>
> **推荐的开发调试方式**：在关键流程处直接用 `PRINT` / `PRINTFORML` 输出所需信息，测试完成后手动删除这些行。简单直接，无需依赖调试模式。

---

下文列出 Emuera 的全部调试功能，供需要了解引擎能力或排查复杂 bug 时参考。

## 启用调试模式

用命令行参数 `-Debug` 启动 Emuera：

1. 右键 `Emuera.exe` → 创建快捷方式
2. 右键快捷方式 → 属性
3. 在「目标」末尾追加 ` -Debug`（注意前面的空格；若路径已在引号中，加在引号之后）

启动后自动创建 `debug/` 文件夹，变量监视列表和调试控制台日志保存于此。

## 调试专用语法

### `;#;` — 调试专用行

```
;#;PRINTFORML DEBUG: MONEY={MONEY}
```

以 `;#;` 开头的行仅在调试模式下执行，非调试模式下视为注释。

### `[IF_DEBUG]` / `[IF_NDEBUG]` — 调试条件块

```
[IF_DEBUG]
    PRINTL 此块仅在调试模式下编译执行
[ENDIF]

[IF_NDEBUG]
    PRINTL 此块仅在非调试模式下编译执行
[ENDIF]
```

以整行为单位，同行不能附带其他代码。

## 调试命令

### DEBUGPRINT 系

| 命令 | 说明 |
|------|------|
| `DEBUGPRINT 字符串` | 输出到调试窗口 |
| `DEBUGPRINTL 字符串` | 输出到调试窗口 + 换行 |
| `DEBUGPRINTFORM 字符串` | 格式化输出到调试窗口 |
| `DEBUGPRINTFORML 字符串` | 格式化输出到调试窗口 + 换行 |

非调试模式下**完全无操作**（连参数解析都不会发生，因此格式错误也不会报错）。这些命令在非调试模式下被视为"无操作指令"而非注释，因此放在 `SIF` 后面也是安全的。输出目标为调试窗口「控制台」标签页。

### DEBUGCLEAR

```
DEBUGCLEAR
```

清空调试窗口所有文字。无需参数。非调试模式下无操作。

### ASSERT

```
ASSERT 表达式
```

表达式为 0 时报错终止；非 0 时无操作。非调试模式下无操作。

## 调试变量

仅调试模式下有意义的值，非调试模式返回 0 或空字符串：

| 变量 | 类型 | 说明 |
|------|------|------|
| `__FILE__` | string | 当前执行的 ERB 文件名 |
| `__LINE__` | int | 当前执行的行号 |
| `__FUNCTION__` | string | 当前执行的函数名 |

## 调试窗口

调试模式下通过菜单「调试」→「调试窗口」或快捷键 `Ctrl+D` 打开，包含三个标签页：

| 标签页 | 功能 |
|--------|------|
| 控制台 | `DEBUGPRINT` 等命令的输出目标；可在此输入调试命令（无需 `@` 前缀，且 MASTER 不会变为「イカサマ」）。**即使「デバッグコマンドを使用する」配置项为 NO 也有效。** |
| 变量监视 | 手动输入变量名或表达式查看实时值；关闭窗口时自动保存列表 |
| 堆栈跟踪 | 当前函数调用链（顶部为当前函数） |

快捷键 `Ctrl+R` 可刷新调试窗口信息（不切换焦点）。

## 旧式 @ 调试命令（非调试模式）

> **注意**：此功能是调试模式出现之前的旧式调试手段。**目前推荐使用调试模式**。此功能需要在 `emuera.config` 中将「デバッグコマンドを使用する」设为 YES（默认为 NO）。

在游戏主控制台中输入以 `@` 开头的命令，可直接执行 ERB 语句：

```
@MONEY = 10000
@PRINTV FLAG:200
@PRINTFORM %NAME:MASTER% 的 CFLAG(1) = {CFLAG:MASTER:1}
@ADDCHARA 1
@ FLAG:200                           ; 不带命令，直接求值并显示
```

无法使用改变流程（`IF`/`CALL`/`INPUT`/`WAIT` 等）的命令。

### 特殊命令

| 命令 | 功能 |
|------|------|
| `@REBOOT` | 重启引擎，重新加载 `emuera.config`、CSV、ERB 文件 |
| `@OUTPUT` | 将当前日志输出到 `emuera.log`（等同于 `OUTPUTLOG`） |
| `@EXIT` | 退出 Emuera（等同于 `QUIT`） |
| `@CONFIG` | 打开设置对话框 |
| `@DEBUG` | 打开调试窗口（仅调试模式有效） |

### 作弊威慑

执行除上述特殊命令外的普通 ERB 命令时，MASTER 的名称和称呼会被强制改为「イカサマ」（作弊者），以防止滥用。
